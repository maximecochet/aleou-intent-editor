Comment utiliser cet éditeur ?
Chaque fichier d'intention décrit un type de demande que l'IA sait gérer. Pour chaque intention, définissez des scénarios — les différentes situations et les emails que l'IA rédige en réponse. Passez la souris sur le ⓘ pour des explications. Cliquez sur Télécharger le JSON quand vous avez terminé.
📋Informations générales▼
iIdentifiant unique de cette intention. Doit être en MAJUSCULES avec des underscores. Ne pas modifier sauf si vous créez une nouvelle intention depuis zéro.
iÉtiquettes de catégorisation. Servent à filtrer et analyser les emails dans les rapports. Appuyez sur Entrée pour valider chaque tag.
iUne ou deux phrases décrivant cette intention. L'IA utilise ce résumé pour comprendre le contexte global avant de choisir un scénario.
💡Exemples de classification▼

Ces exemples servent à entraîner l'IA à reconnaître cette intention dans un email.

iMots ou expressions courts qui, s'ils apparaissent dans un email, suggèrent fortement cette intention. Appuyez sur Entrée pour valider.

Appuyez sur Entrée pour ajouter

iExemples de phrases complètes qu'un client ou prestataire pourrait écrire pour exprimer cette intention.
🗂️Scénarios▼

Un scénario = une situation particulière et la réponse appropriée. Le dernier scénario doit toujours être le "Fallback".

⚖️Règles de sélection du scénario▼

Ces règles expliquent à l'IA comment choisir le bon scénario. Référencez les scénarios par leur clé.

iDécrit la logique de décision lorsque l'email vient de l'organisateur. Listez les conditions dans l'ordre de priorité.
iDécrit la logique de décision lorsque l'email vient du lieu ou du prestataire. Si aucun email prestataire n'est prévu, indiquez-le explicitement.
⚙️Paramètres avancés▼
iIndique à l'IA quels champs du dossier API elle doit consulter. Utilisez la notation pointée : objet.sous_champ (ex: brief.catering).
Champ du dossierPourquoi l'IA en a besoin
iVariables que l'IA doit extraire du dossier pour construire la réponse. Ces variables sont injectées dans les emails générés sous la forme [NOM_VARIABLE].

Appuyez sur Entrée pour ajouter. Variables utilisées dans les emails : ■ vert = OK, ■ rouge = non utilisée.
